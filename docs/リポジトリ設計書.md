# リポジトリ設計書：airc

## 1. リポジトリ構造概要

### 1.1 全体構造

```
airc/
├── .gitignore
├── README.md
├── LICENSE
├── package.json
├── bin/
│   └── cli.js
└── projects/
    ├── default/       ← 初期プロジェクト（必須）
    │   ├── .github/
    │   │   ├── chatmodes/
    │   │   └── prompts/
    │   ├── .claude/
    │   └── CLAUDE.md
    ├── projectA/      ← カスタムプロジェクト（追加可能）
    │   └── (同様の構造)
    └── projectB/
        └── (同様の構造)
```

### 1.2 プロジェクトディレクトリの役割

`projects/` 配下には、複数のプロジェクト設定を格納します。各プロジェクトは独立した設定セットを持ちます。

- **default**: 初期から用意されるデフォルトプロジェクト（プロジェクト指定なしの場合に使用）
- **projectA, projectB, ...**: 後から追加できるカスタムプロジェクト

## 2. プロジェクト内部構造

### 2.1 ディレクトリ構成

各プロジェクトディレクトリは以下の構造を持ちます：

```
projects/{project-name}/
├── .github/
│   ├── chatmodes/          # GitHub Copilot Chat modes
│   │   ├── file1.md
│   │   └── file2.md
│   └── prompts/            # GitHub Copilot Prompts
│       ├── file1.md
│       └── file2.md
├── .claude/                # Claude AI 設定
│   ├── instructions/
│   │   └── *.md
│   └── README.md
└── CLAUDE.md               # Claude ルート設定ファイル
```

### 2.2 各ディレクトリの説明

| ディレクトリ          | 用途                                    | 対応AIツール       |
| --------------------- | --------------------------------------- | ------------------ |
| `.github/chatmodes/`  | GitHub Copilot のチャットモード設定     | GitHub Copilot     |
| `.github/prompts/`    | GitHub Copilot のプロンプト設定         | GitHub Copilot     |
| `.claude/`            | Claude AI の詳細設定                    | Claude             |
| ルート（`CLAUDE.md`） | Claude AI のルート設定ファイル（単一） | Claude             |

## 3. GitHub API 設計

### 3.1 ファイルリスト取得

GitHub API を使用してプロジェクト配下のファイルを動的に取得します：

```
GitHub API エンドポイント:
https://api.github.com/repos/maru3460/airc/contents/projects/{project}

リクエスト例:
GET https://api.github.com/repos/maru3460/airc/contents/projects/default
```

レスポンス例（一部抜粋）：
```json
[
  {
    "name": ".github",
    "path": "projects/default/.github",
    "type": "dir"
  },
  {
    "name": "CLAUDE.md",
    "path": "projects/default/CLAUDE.md",
    "type": "file"
  }
]
```

### 3.2 ファイルダウンロード URL

実ファイルは raw.githubusercontent.com からダウンロードします：

```
ベースURL: https://raw.githubusercontent.com/maru3460/airc/main

URL 構造:
https://raw.githubusercontent.com/maru3460/airc/main/{GitHub APIで取得したpath}

例:
https://raw.githubusercontent.com/maru3460/airc/main/projects/default/.github/chatmodes/best-practices.md
https://raw.githubusercontent.com/maru3460/airc/main/projects/default/.claude/instructions/basic.md
https://raw.githubusercontent.com/maru3460/airc/main/projects/default/CLAUDE.md
```

## 4. ローカルパス設計

### 4.1 ダウンロード先パス構造

CLI 実行ディレクトリをルートとして、以下のパスに配置されます：

```
作業ディレクトリ/
├── .github/
│   ├── chatmodes/{filename}
│   └── prompts/{filename}
├── .claude/{filename}
└── {filename}  (ルートファイル)
```

### 4.2 パス変換ルール

GitHub リポジトリのパスからローカルパスへの変換：

```
GitHub パス:
projects/default/.github/chatmodes/file.md

↓ projects/{project}/ を除去

ローカルパス:
.github/chatmodes/file.md
```

変換例：

| GitHub パス                                            | ローカルパス                         |
| ------------------------------------------------------ | ------------------------------------ |
| `projects/default/.github/chatmodes/file.md`           | `.github/chatmodes/file.md`          |
| `projects/default/.github/prompts/refactor.md`         | `.github/prompts/refactor.md`        |
| `projects/default/.claude/instructions/basic.md`       | `.claude/instructions/basic.md`      |
| `projects/default/CLAUDE.md`                           | `CLAUDE.md`                          |

## 5. プロジェクト管理

### 5.1 プロジェクト追加方法

新しいプロジェクトを追加する手順：

1. `projects/` 配下に新しいプロジェクトディレクトリを作成
   ```bash
   mkdir -p projects/projectA
   ```

2. 必要なディレクトリ構造を作成
   ```bash
   mkdir -p projects/projectA/.github/chatmodes
   mkdir -p projects/projectA/.github/prompts
   mkdir -p projects/projectA/.claude
   ```

3. 設定ファイルを配置
   - `.github/chatmodes/` 配下に Copilot chatmodes ファイルを配置
   - `.github/prompts/` 配下に Copilot prompts ファイルを配置
   - `.claude/` 配下に Claude 設定ファイルを配置
   - ルートに `CLAUDE.md` を配置（必要に応じて）

4. GitHub にプッシュ
   ```bash
   git add projects/projectA
   git commit -m "Add projectA configuration"
   git push
   ```

**重要**: GitHub API で動的にファイルリストを取得するため、コード側の修正は不要です。リポジトリにファイルを追加するだけで、自動的に認識されます。

### 5.2 プロジェクト命名規則

- **小文字とハイフン推奨**: `project-a`, `frontend-app` など
- **英数字とハイフンのみ**: 特殊文字やスペースは避ける
- **説明的な名前**: プロジェクトの用途が分かる名前を付ける

## 6. ファイル命名規則

### 6.1 推奨命名規則

- **小文字とハイフン**: ファイル名は小文字とハイフンを使用（例: `react-best-practices.md`）
- **説明的な名前**: ファイルの内容が分かる名前を付ける
- **拡張子**: Markdown ファイルは `.md` を使用

### 6.2 特殊ファイル

- `CLAUDE.md`: Claude AI のルート設定ファイル（固定名）

## 8. セキュリティ考慮事項

### 8.1 アクセス制御

- リポジトリは public である必要があります（`raw.githubusercontent.com` 経由でアクセスするため）
- 機密情報をリポジトリに含めないこと

### 8.2 ファイル検証

- プロジェクト名の検証（GitHub API で 404 が返る場合はエラー）
- ファイル名のバリデーションにより、ディレクトリトラバーサルを防止
- パスインジェクション対策（`../` などの不正なパスを拒否）
