# ローカルプロファイル機能 - 使用例

## コマンド仕様

### `airc init`

カレントディレクトリに `.airc/` を作成し、初期化します。

**実行内容**:
1. `.airc/` ディレクトリ作成
2. `config.json` 作成：`{"current": "main"}`
3. `.sync` 作成（デフォルトパターン）
4. `profiles/main/` ディレクトリ作成（空）

**使用例**:
```bash
cd my-project
airc init
```

**出力例**:
```
✨ .airc/ を初期化しました
📝 デフォルトプロファイル「main」を作成しました
💡 次のステップ：
   - .airc/.sync を編集して同期対象を設定
   - airc new <name> で新しいプロファイルを作成
```

**エラーケース**:
- すでに `.airc/` が存在する場合
  ```
  ❌ エラー：.airc/ はすでに存在します
  ```

---

### `airc list`

ローカルプロファイルの一覧を表示します。

**使用例**:
```bash
airc list
```

**出力例**:
```
✨ ローカルプロファイル：
  * main (active)
    feature-x
    tmp
```

**エラーケース**:
- `.airc/` が存在しない場合
  ```
  ❌ エラー：.airc/ が初期化されていません
  まず `airc init` を実行してください
  ```

---

### `airc new [profilename]`

新規プロファイルを作成し、アクティブプロファイルに切り替えます。

**実行内容**:
1. **実ファイル → 旧アクティブプロファイルに保存**（退避）
2. `profiles/[profilename]/` を作成
3. **実ファイル → 新プロファイルにコピー**
4. アクティブを `[profilename]` に切り替え（`config.json` 更新）
5. **実ファイルは変化なし**

**使用例**:
```bash
# main がアクティブな状態で
airc new feature-x
```

**出力例**:
```
💾 現在のプロファイル「main」を保存しました
✨ 新しいプロファイル「feature-x」を作成しました
🔄 アクティブプロファイルを「feature-x」に切り替えました
```

**エラーケース**:
- プロファイル名がすでに存在する場合
  ```
  ❌ エラー：プロファイル「feature-x」はすでに存在します
  ```
- プロファイル名が不正な場合（空、特殊文字など）
  ```
  ❌ エラー：プロファイル名が不正です
  使用できる文字：英数字、ハイフン、アンダースコア
  ```

---

### `airc use [profilename]`

既存のプロファイルに切り替えます。

**実行内容**:
1. **実ファイル → 旧アクティブプロファイルに保存**（退避）
2. `profiles/[profilename]/` → 実ファイルに展開
3. `config.json` を更新：`{"current": "[profilename]"}`

**使用例**:
```bash
airc use feature-x
```

**出力例**:
```
💾 現在のプロファイル「main」を保存しました
🔄 プロファイル「feature-x」に切り替えました
✨ 3 ファイルを展開しました
```

**エラーケース**:
- プロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「non-existing」が見つかりません
  利用可能なプロファイル：
    * main (active)
      feature-x
  ```
- すでにアクティブなプロファイルを指定した場合
  ```
  ℹ️  プロファイル「main」はすでにアクティブです
  ```

---

### `airc rename [old] [new]`

プロファイルをリネームします。

**実行内容**:
1. `profiles/[old]/` → `profiles/[new]/` にリネーム
2. アクティブプロファイルが `[old]` だった場合、`config.json` も更新

**使用例**:
```bash
airc rename old-name new-name
```

**出力例**:
```
✨ プロファイル「old-name」を「new-name」にリネームしました
```

**エラーケース**:
- 元のプロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「old-name」が見つかりません
  ```
- 新しい名前がすでに存在する場合
  ```
  ❌ エラー：プロファイル「new-name」はすでに存在します
  ```

---

### `airc delete [profilename]`

プロファイルを削除します。

**実行内容**:
1. 確認ダイアログを表示
2. `profiles/[profilename]/` を削除

**使用例**:
```bash
airc delete feature-x
```

**出力例**:
```
⚠️  プロファイル「feature-x」を削除します
この操作は取り消せません。本当に削除しますか？ (y/N): y
✨ プロファイル「feature-x」を削除しました
```

**エラーケース**:
- アクティブプロファイルを削除しようとした場合
  ```
  ❌ エラー：アクティブなプロファイル「main」は削除できません
  別のプロファイルに切り替えてから削除してください
  ```
- プロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「non-existing」が見つかりません
  ```

---

### `airc clear`

現在のアクティブプロファイルの実ファイル（`.sync` 対象）を削除します。

**実行内容**:
1. `.sync` で定義されたファイルのリストを取得
2. 確認ダイアログを表示
3. 実ファイルを削除
4. **プロファイル側（`.airc/profiles/`）は削除しない**

**使用例**:
```bash
airc clear
```

**出力例**:
```
⚠️  現在のプロファイル「main」の以下のファイルを削除します：
  - .github/chatmodes/file1.md
  - .github/chatmodes/file2.md
  - .claude/instructions.md
  - CLAUDE.md
本当に削除しますか？ (y/N): y
✨ 4 ファイルを削除しました
```

**注意事項**:
- `.sync` に含まれないファイルは削除されません
  - 例：`.sync` に `.github/chatmodes/` のみ定義されている場合、`.github/prompts/` は削除されない

---

### `airc sync [profile]`

GitHub リポジトリからプロファイルをダウンロードし、ローカルプロファイルとして追加します。

**実行内容**:
1. GitHub から `profiles/[profile]/` の内容をダウンロード
2. `.airc/profiles/[profile]/` に保存
3. アクティブプロファイルを `[profile]` に切り替え
4. `profiles/[profile]/` の内容をカレントディレクトリに展開

**使用例**:
```bash
airc sync main
airc sync main --force  # 既存ファイルを強制上書き
```

**出力例**:
```
📥 GitHub からプロファイル「main」をダウンロード中...
✨ 8 ファイルをダウンロードしました
💾 現在のプロファイル「main」を保存しました
🔄 プロファイル「main」に切り替えました
✨ 8 ファイルを展開しました
```

**エラーケース**:
- プロファイル名がすでに存在する場合（重要！）
  ```
  ❌ エラー：プロファイル「main」はすでに存在します
  別の名前を使うか、`airc delete main` で削除してから再実行してください
  ```
- GitHub 上にプロファイルが存在しない場合
  ```
  ❌ エラー：リモートプロファイル「non-existing」が見つかりません
  利用可能なプロファイル：
    - main
    - airc
  ```

**オプション**:
- `--force`, `-f`: 既存ファイルを確認なしで上書き
- `--list`, `-l`: GitHub 上の利用可能なプロファイル一覧を表示

---

### `airc sync --list`

GitHub リポジトリ上のプロファイル一覧を表示します。

**使用例**:
```bash
airc sync --list
airc sync -l
```

**出力例**:
```
✨ リモートプロファイル（maru3460/airc）：
  - main (default)
  - airc
  - example
```

**エラーケース**:
- GitHub API エラー
  ```
  ❌ エラー：GitHub API へのアクセスに失敗しました
  ネットワーク接続を確認してください
  ```
