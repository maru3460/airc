# ローカルプロファイル機能 - 概要

## 概要

airc をローカルメインのプロファイル管理ツールに変更します。各プロジェクトの `.airc/` ディレクトリでプロファイルを管理し、Git ブランチのように切り替えられる機能を提供します。

## コマンド一覧

### ローカルプロファイル管理（メイン機能）
- `airc init` - プロジェクトの初期化
- `airc list` - ローカルプロファイル一覧
- `airc new [name]` - 新規プロファイル作成
- `airc use [name]` - プロファイル切り替え
- `airc rename [old] [new]` - プロファイルリネーム
- `airc delete [name]` - プロファイル削除
- `airc clear` - 実ファイル削除

### リモート同期（補助機能）
- `airc sync [profile]` - GitHub からプロファイルをダウンロード
- `airc sync --list` - GitHub 上のプロファイル一覧

## 基本コンセプト

- **ローカルファイルが主体**：カレントディレクトリの `.airc/` 配下でプロファイル管理
- **プロジェクト単位**：複数ディレクトリから同じ場所を触らない（プロジェクトごとに `.airc/` を持つ）
- **リモートは補助**：GitHub はあくまで追加のソース（既存の `airc sync` コマンドでダウンロード可能）

## ディレクトリ構造

```
project/
├── .airc/
│   ├── config.json        # {"current": "main"}
│   ├── .sync              # 同期対象パターン定義
│   └── profiles/
│       ├── main/          # デフォルトプロファイル（init時に作成）
│       │   ├── .github/
│       │   │   ├── chatmodes/
│       │   │   └── prompts/
│       │   ├── .claude/
│       │   └── CLAUDE.md
│       └── feature-x/     # 追加プロファイル
│           ├── .github/
│           ├── .claude/
│           └── CLAUDE.md
├── .github/               # 実際に使われるファイル（アクティブプロファイルから展開）
├── .claude/
└── CLAUDE.md
```

**注**: 既存の `airc sync` コマンドはリモート（GitHub）からのダウンロード機能として引き続き利用可能。
