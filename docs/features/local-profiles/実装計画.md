# ローカルプロファイル機能 - 実装計画

### Phase 1: 既存実装の確認と準備 🔍

**目的**: 既存実装を確認し、新機能追加の準備を行う（破壊的変更を早期に実施）

**実装内容**:

1. **既存コードの確認**
   - `src/cli/commands/sync.ts`: リモート同期コマンド（そのまま活用）
   - `src/cli/commands/list.ts`: リモート一覧コマンド（Phase 1 で削除し、sync に統合）
   - `src/utils/fs.ts`, `src/utils/path.ts`: ファイル操作（ローカルプロファイルでも再利用）
   - `src/api/`: GitHub API 関連（Phase 5 で活用）

2. **型定義の追加** (`src/types.ts`)
   ```typescript
   // 既存の型定義に追加
   export interface LocalConfig {
     current: string;  // アクティブプロファイル名
   }

   export interface SyncPattern {
     include: string[];
     exclude: string[];
   }
   ```

3. **定数の追加** (`src/config.ts`)
   ```typescript
   // ローカルプロファイル関連の定数を追加
   export const AIRC_DIR = '.airc';
   export const PROFILES_DIR = 'profiles';
   export const CONFIG_FILE = 'config.json';
   export const SYNC_FILE = '.sync';
   export const DEFAULT_PROFILE = 'main';
   ```

4. **`airc sync --list` の実装** (`src/cli/commands/sync.ts`)
   - 既存の `list.ts` の機能を `sync.ts` に統合
   - `--list` オプションでリモートプロファイル一覧を表示
   - `list.ts` を削除
   - **破壊的変更**: `airc list` コマンドを削除（Phase 4 でローカルプロファイル一覧として再実装）

**成果物**:
- 新しい型定義と定数
- `airc sync --list` コマンド（リモートプロファイル一覧）
- `airc list` コマンドの削除（次の Phase でローカルプロファイル一覧として実装）

**テスト項目**:

### Phase 3: ファイル同期機能 🔄

**目的**: プロファイルと実ファイル間の同期機能を実装

**実装内容**:

1. **ファイル同期** (`src/utils/syncFiles.ts`)
   - `.sync` パターンに基づくファイルリスト取得
   - 実ファイル → プロファイルへのコピー（保存）
   - プロファイル → 実ファイルへのコピー（展開）
   - ファイル削除（clear 用）
   ```typescript
   export async function saveToProfile(profileName: string, patterns: SyncPattern): Promise<void>
   export async function restoreFromProfile(profileName: string): Promise<void>
   export async function clearFiles(patterns: SyncPattern): Promise<void>
   ```

2. **既存ユーティリティの活用**
   - `src/utils/path.ts`: パス検証（既存の `isValidPath` を活用）
   - `src/utils/fs.ts`: ファイル操作（既存の `ensureDir`, `saveFile` を活用）

3. **エラーハンドリング**
   - ファイル読み書きエラー
   - パーミッションエラー
   - 存在しないプロファイルエラー

**成果物**:
- ファイル同期機能
- 安全なパス操作（既存機能を活用）

### Phase 5: リモート連携機能の統合 🌐

**目的**: 既存の `airc sync` 機能をローカルプロファイルシステムと統合

**実装内容**:

1. **`airc sync` コマンドの拡張** (`src/cli/commands/sync.ts`)
   - ダウンロード先を `.airc/profiles/[profile]/` に変更
   - ダウンロード後、アクティブプロファイルに設定
   - 実ファイルへの展開
   - `--list` オプションは Phase 1 で実装済み

2. **プロファイル統合**
   - GitHub ダウンロード → `.airc/profiles/[profile]/` に保存
   - アクティブプロファイル切り替え
   - 実ファイルへの展開

**成果物**:
- `airc sync [profile]`（`.airc/profiles/` にダウンロード）
- `airc sync --list`（Phase 1 で実装済み）
- `airc list`（Phase 4 で実装済み）
- `airc delete [name]`（Phase 4 で実装済み）

**テスト項目**:
- [ ] `airc sync --list` による一覧表示（Phase 1 実装済み）
- [ ] `airc sync [profile]` で `.airc/profiles/` にダウンロード
- [ ] ダウンロード後のプロファイル切り替え
- [ ] `airc list` がローカルプロファイル一覧を表示（Phase 4 実装済み）

**完了条件**:

- ✅ ダウンロード後のプロファイル切り替えが正しく動作する
- ✅ ローカルプロファイルとリモートプロファイルが統合される
