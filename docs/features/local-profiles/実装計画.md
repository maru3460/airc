# ローカルプロファイル機能 - 実装計画

## 概要

airc をローカルメインのプロファイル管理ツールに変更します。各プロジェクトの `.airc/` ディレクトリでプロファイルを管理し、Git ブランチのように切り替えられる機能を提供します。

## 基本コンセプト

- **ローカルファイルが主体**：カレントディレクトリの `.airc/` 配下でプロファイル管理
- **プロジェクト単位**：複数ディレクトリから同じ場所を触らない（プロジェクトごとに `.airc/` を持つ）
- **リモートは補助**：GitHub はあくまで追加のソース（既存リポジトリからダウンロード可能）

## ディレクトリ構造

```
project/
├── .airc/
│   ├── config.json        # {"current": "main"}
│   ├── .sync              # 同期対象パターン定義
│   └── profiles/
│       ├── main/          # デフォルトプロファイル（init時に作成）
│       │   ├── .github/
│       │   │   ├── chatmodes/
│       │   │   └── prompts/
│       │   ├── .claude/
│       │   └── CLAUDE.md
│       └── feature-x/     # 追加プロファイル
│           ├── .github/
│           ├── .claude/
│           └── CLAUDE.md
├── .github/               # 実際に使われるファイル（アクティブプロファイルから展開）
├── .claude/
└── CLAUDE.md
```

## `.sync` ファイル仕様

`.airc/.sync` は同期対象ファイルのパターンを定義します。`.gitignore` の簡素版のような記法です。

### 記法

```
# .airc/.sync の例

# ディレクトリ指定（再帰的に全ファイル対象）
.github/chatmodes/
.github/prompts/
.claude/

# ファイル指定
CLAUDE.md

# ワイルドカード
*.md

# 除外記法（! プレフィックス）
!.github/chatmodes/draft.md
```

### デフォルトパターン

`airc init` 実行時に以下のパターンで `.sync` を自動生成：

```
.github/
.claude/
CLAUDE.md
```

## コマンド仕様

### `airc init`

カレントディレクトリに `.airc/` を作成し、初期化します。

**実行内容**:
1. `.airc/` ディレクトリ作成
2. `config.json` 作成：`{"current": "main"}`
3. `.sync` 作成（デフォルトパターン）
4. `profiles/main/` ディレクトリ作成（空）

**使用例**:
```bash
cd my-project
airc init
```

**出力例**:
```
✨ .airc/ を初期化しました
📝 デフォルトプロファイル「main」を作成しました
💡 次のステップ：
   - .airc/.sync を編集して同期対象を設定
   - airc new <name> で新しいプロファイルを作成
```

**エラーケース**:
- すでに `.airc/` が存在する場合
  ```
  ❌ エラー：.airc/ はすでに存在します
  ```

---

### `airc list`

ローカルプロファイルの一覧を表示します。

**使用例**:
```bash
airc list
```

**出力例**:
```
✨ ローカルプロファイル：
  * main (active)
    feature-x
    tmp
    default
```

**エラーケース**:
- `.airc/` が存在しない場合
  ```
  ❌ エラー：.airc/ が初期化されていません
  まず `airc init` を実行してください
  ```

---

### `airc new [profilename]`

新規プロファイルを作成し、アクティブプロファイルに切り替えます。

**実行内容**:
1. **実ファイル → 旧アクティブプロファイルに保存**（退避）
2. `profiles/[profilename]/` を作成
3. **実ファイル → 新プロファイルにコピー**
4. アクティブを `[profilename]` に切り替え（`config.json` 更新）
5. **実ファイルは変化なし**

**使用例**:
```bash
# main がアクティブな状態で
airc new feature-x
```

**出力例**:
```
💾 現在のプロファイル「main」を保存しました
✨ 新しいプロファイル「feature-x」を作成しました
🔄 アクティブプロファイルを「feature-x」に切り替えました
```

**エラーケース**:
- プロファイル名がすでに存在する場合
  ```
  ❌ エラー：プロファイル「feature-x」はすでに存在します
  ```
- プロファイル名が不正な場合（空、特殊文字など）
  ```
  ❌ エラー：プロファイル名が不正です
  使用できる文字：英数字、ハイフン、アンダースコア
  ```

---

### `airc use [profilename]`

既存のプロファイルに切り替えます。

**実行内容**:
1. **実ファイル → 旧アクティブプロファイルに保存**（退避）
2. `profiles/[profilename]/` → 実ファイルに展開
3. `config.json` を更新：`{"current": "[profilename]"}`

**使用例**:
```bash
airc use feature-x
```

**出力例**:
```
💾 現在のプロファイル「main」を保存しました
🔄 プロファイル「feature-x」に切り替えました
✨ 3 ファイルを展開しました
```

**エラーケース**:
- プロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「non-existing」が見つかりません
  利用可能なプロファイル：
    * main (active)
      feature-x
  ```
- すでにアクティブなプロファイルを指定した場合
  ```
  ℹ️  プロファイル「main」はすでにアクティブです
  ```

---

### `airc rename [old] [new]`

プロファイルをリネームします。

**実行内容**:
1. `profiles/[old]/` → `profiles/[new]/` にリネーム
2. アクティブプロファイルが `[old]` だった場合、`config.json` も更新

**使用例**:
```bash
airc rename old-name new-name
```

**出力例**:
```
✨ プロファイル「old-name」を「new-name」にリネームしました
```

**エラーケース**:
- 元のプロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「old-name」が見つかりません
  ```
- 新しい名前がすでに存在する場合
  ```
  ❌ エラー：プロファイル「new-name」はすでに存在します
  ```

---

### `airc delete [profilename]`

プロファイルを削除します。

**実行内容**:
1. 確認ダイアログを表示
2. `profiles/[profilename]/` を削除

**使用例**:
```bash
airc delete feature-x
```

**出力例**:
```
⚠️  プロファイル「feature-x」を削除します
この操作は取り消せません。本当に削除しますか？ (y/N): y
✨ プロファイル「feature-x」を削除しました
```

**エラーケース**:
- アクティブプロファイルを削除しようとした場合
  ```
  ❌ エラー：アクティブなプロファイル「main」は削除できません
  別のプロファイルに切り替えてから削除してください
  ```
- プロファイルが存在しない場合
  ```
  ❌ エラー：プロファイル「non-existing」が見つかりません
  ```

---

### `airc clear`

現在のアクティブプロファイルの実ファイル（`.sync` 対象）を削除します。

**実行内容**:
1. `.sync` で定義されたファイルのリストを取得
2. 確認ダイアログを表示
3. 実ファイルを削除
4. **プロファイル側（`.airc/profiles/`）は削除しない**

**使用例**:
```bash
airc clear
```

**出力例**:
```
⚠️  現在のプロファイル「main」の以下のファイルを削除します：
  - .github/chatmodes/file1.md
  - .github/chatmodes/file2.md
  - .claude/instructions.md
  - CLAUDE.md
本当に削除しますか？ (y/N): y
✨ 4 ファイルを削除しました
```

**注意事項**:
- `.sync` に含まれないファイルは削除されません
  - 例：`.sync` に `.github/chatmodes/` のみ定義されている場合、`.github/prompts/` は削除されない

---

### `airc remote dl [profile]`

GitHub リポジトリからプロファイルをダウンロードし、ローカルプロファイルとして追加します。

**実行内容**:
1. GitHub から `profiles/[profile]/` の内容をダウンロード
2. `.airc/profiles/[profile]/` に保存
3. アクティブプロファイルを `[profile]` に切り替え
4. `profiles/[profile]/` の内容をカレントディレクトリに展開

**使用例**:
```bash
airc remote dl default
```

**出力例**:
```
📥 GitHub からプロファイル「default」をダウンロード中...
✨ 8 ファイルをダウンロードしました
💾 現在のプロファイル「main」を保存しました
🔄 プロファイル「default」に切り替えました
✨ 8 ファイルを展開しました
```

**エラーケース**:
- プロファイル名がすでに存在する場合（重要！）
  ```
  ❌ エラー：プロファイル「default」はすでに存在します
  別の名前を使うか、`airc delete default` で削除してから再実行してください
  ```
- GitHub 上にプロファイルが存在しない場合
  ```
  ❌ エラー：リモートプロファイル「non-existing」が見つかりません
  利用可能なプロファイル：
    - default
    - airc
  ```

## 既存機能との統合

### 既存の `airc` コマンド（引数なし）の扱い

現在の `airc` コマンド（`airc`, `airc -p profile`, `airc --force`）は**廃止**し、新しいコマンド体系に完全移行します。

**移行ガイド**:

| 旧コマンド | 新コマンド |
|-----------|-----------|
| `airc` | `airc use main` または `airc remote dl default` |
| `airc -p my-profile` | `airc remote dl my-profile` |
| `airc --force` | （不要：常に上書き） |

### マイグレーションスクリプト

既存ユーザー向けに、初回実行時にマイグレーションガイドを表示：

```typescript
// src/cli.ts
async function checkMigration() {
  const hasOldUsage = !hasAircDirectory() && process.argv.length === 2;

  if (hasOldUsage) {
    console.log('👋 airc v2.0 へようこそ！');
    console.log('');
    console.log('📝 使い方が変わりました：');
    console.log('   1. airc init               # プロジェクトを初期化');
    console.log('   2. airc remote dl default  # GitHub からダウンロード');
    console.log('   3. airc list               # プロファイル一覧表示');
    console.log('');
    console.log('💡 詳細: https://github.com/maru3460/airc#readme');
    console.log('');
  }
}
```

## 開発時の注意事項

### セキュリティ

1. **パストラバーサル防止**
   ```typescript
   function validatePath(path: string): boolean {
     // "../" を含むパスを拒否
     if (path.includes('..')) {
       return false;
     }
     // 絶対パスを拒否
     if (path.startsWith('/')) {
       return false;
     }
     return true;
   }
   ```

2. **ファイル操作の安全性**
   - 書き込み先はカレントディレクトリ配下のみ
   - シンボリックリンクの扱いに注意

### エラーハンドリング

すべてのファイル操作で適切なエラーハンドリング：

```typescript
try {
  await fs.writeFile(path, content);
} catch (error) {
  if (error.code === 'EACCES') {
    throw new Error(`❌ エラー：ファイル "${path}" への書き込み権限がありません`);
  }
  throw error;
}
```
