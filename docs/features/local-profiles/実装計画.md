### Phase 5: リモート連携機能の統合 🌐

**目的**: 既存の `airc sync`, `airc list` 機能をローカルプロファイルシステムと統合（破壊的変更）

**実装内容**:

1. **`airc sync` コマンドの拡張** (`src/cli/commands/sync.ts`)
   - ダウンロード先を `.airc/profiles/[profile]/` に変更
   - ダウンロード後、アクティブプロファイルに設定
   - 実ファイルへの展開
   - `--list` オプションの追加（既存の `list` コマンド機能を統合）

2. **`airc list` コマンドの置き換え**
   - 既存の `list.ts` を `listRemote.ts` にリネーム
   - `localList.ts` を `list.ts` にリネーム
   - `airc list` → ローカルプロファイル一覧
   - `airc sync --list` → リモートプロファイル一覧

3. **`airc delete` コマンドの置き換え**
   - `localDelete.ts` を `delete.ts` にリネーム
   - `airc delete [name]` → ローカルプロファイル削除

4. **プロファイル統合**
   - GitHub ダウンロード → `.airc/profiles/[profile]/` に保存
   - アクティブプロファイル切り替え
   - 実ファイルへの展開

**成果物**:
- `airc sync [profile]`（`.airc/profiles/` にダウンロード）
- `airc sync --list`（リモートプロファイル一覧）
- `airc list`（ローカルプロファイル一覧に変更）
- `airc delete [name]`（ローカルプロファイル削除）

## 開発時の注意事項

### エラーハンドリング

すべてのファイル操作で適切なエラーハンドリング：

```typescript
try {
  await fs.writeFile(path, content);
} catch (error) {
  if (error.code === 'EACCES') {
    throw new Error(`❌ エラー：ファイル "${path}" への書き込み権限がありません`);
  }
  throw error;
}
```

### コミットメッセージ

```
<type>(local-profiles): <subject>

<body>

<footer>
```

**例**:
```
feat(local-profiles): Add profile switching functionality

- Implement `airc use` command
- Add file synchronization between profiles
- Add save/restore logic

Closes #456
```
