# ローカルプロファイル機能 - 設計

## `.sync` ファイル仕様

`.airc/.sync` は同期対象ファイルのパターンを定義します。`.gitignore` の簡素版のような記法です。

### 記法

```
# .airc/.sync の例

# ディレクトリ指定（再帰的に全ファイル対象）
.github/chatmodes/
.github/prompts/
.claude/

# ファイル指定
CLAUDE.md

# ワイルドカード
*.md

# 除外記法（! プレフィックス）
!.github/chatmodes/draft.md
```

### デフォルトパターン

`airc init` 実行時に以下のパターンで `.sync` を自動生成：

```
.github/
.claude/
CLAUDE.md
```

## 既存機能との統合

### コマンド体系の変更

既存の `airc` コマンドは以下のように変更されます：

| 既存コマンド | 新コマンド | 説明 |
|------------|-----------|------|
| `airc sync` | `airc sync [profile]` | GitHub からプロファイルをダウンロード（既存機能を維持） |
| `airc sync -p profile` | `airc sync [profile]` | プロファイル指定方法を統一 |
| `airc sync --force` | `airc sync [profile] --force` | 強制上書きオプション（既存機能を維持） |
| `airc list` | `airc sync --list` | リモートプロファイル一覧 |
| - | `airc list` | **新機能**: ローカルプロファイル一覧 |

### マイグレーションスクリプト

既存ユーザー向けに、初回実行時にマイグレーションガイドを表示：

```typescript
// src/cli.ts または適切な場所
async function checkMigration() {
  const hasOldUsage = !hasAircDirectory();

  if (hasOldUsage) {
    console.log('👋 airc v2.0 へようこそ！');
    console.log('');
    console.log('📝 新機能が追加されました：');
    console.log('   1. airc init               # プロジェクトを初期化');
    console.log('   2. airc sync main          # GitHub からダウンロード');
    console.log('   3. airc list               # ローカルプロファイル一覧表示');
    console.log('');
    console.log('💡 詳細: https://github.com/maru3460/airc#readme');
    console.log('');
  }
}
```

### 破壊的変更

- `airc list` の挙動が変更されます
  - 旧: GitHub 上のリモートプロファイル一覧
  - 新: ローカルプロファイル一覧
  - 移行: リモート一覧は `airc sync --list` を使用

### パストラバーサル防止

```typescript
function validatePath(path: string): boolean {
  // "../" を含むパスを拒否
  if (path.includes('..')) {
    return false;
  }
  // 絶対パスを拒否
  if (path.startsWith('/')) {
    return false;
  }
  return true;
}
```
